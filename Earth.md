## 总体规划：分五个里程碑与智能体沟通

[GitHub - google/langextract: A Python library for extracting structured information from unstructured text using LLMs with precise source grounding and interactive visualization. · GitHub](https://github.com/google/langextract)

我们将目标拆成五个**里程碑**，每次对话只完成一个里程碑，并将上一个里程碑的输出作为下一个的输入。

### 里程碑地图

1. **需求定义与提取设计** — 让智能体理解你的“星球”要展示什么，并设计出 LangExtract 的提取模板。

2. **跑通 LangExtract，拿到原始数据** — 让智能体教你运行提取，生成带精确溯源的 JSONL 文件。

3. **将提取结果转化为图数据** — 让智能体写一个转换脚本，把实体和关系变成节点和连线。

4. **3D 星球可视化核心** — 让智能体写出可旋转球体、点击展开、无限递归的交互网页。

5. **装配、测试与美化** — 整合全部组件，调整颜色、样式，让星球真正活起来。

---

## 里程碑 1：需求定义与提取设计

**目标：** 和智能体一起敲定“宇宙蓝图”，输出一个清晰的提取配置。

**沟通指令：**

> 你好，我正在构建一个“文本知识星球”。我会提供一段（或多段）文本，需要从中提取各类实体和它们之间的因果关系，最终在一个可旋转的 3D 球体上展示，点击节点能无限展开关联。
> 
> 第一步，请你帮我设计 LangExtract 的提取模板。我的要求是：
> 
> - 提取尽可能丰富的实体类别，比如：人物、动物、物体、地点、事件、情绪、概念等，并且每个实体可以附带属性（如性别、颜色、大小、状态等）。
> 
> - 提取实体间的**因果关系**，例如“A 放飞蝴蝶 导致 蝴蝶扇动翅膀”、“蝴蝶扇动翅膀 导致 远处起风”。关系要求以“(因) 实体A —[关系描述]→ (果) 实体B”的形式表达。
> 
> - 所有提取必须精确锚定原文位置（利用 LangExtract 的溯源能力）。
> 
> - 输出格式要方便我后期构建图网络。
> 
> 请基于以上，为我设计一个详细的 prompt_description 和几个示例 examples（需要你构造几个虚构的简短文本示例）。请用 Python 代码块展示如何定义这些变量。

**你需提前准备：** 一段你想挖掘的文本，比如一篇短篇小说、一篇维基百科文章，或者先用《罗密欧与朱丽叶》选段。你可以直接把文本粘贴给智能体。

**里程碑 1 的成功输出：** 智能体给你一份设计文档，包括：

- `prompt_description` 字符串

- 3~5 个 `ExampleData` 对象

- 说明实体类别和关系类别的定义

## 里程碑 2：跑通 LangExtract，拿到原始数据

**目标：** 在智能体的手把手指导下，实际运行一次提取，获得带有精确位置的 JSONL 结果。

**沟通指令：**

> 很好，现在请根据你上一步设计的模板，帮我写出完整的 LangExtract 提取代码。  
> 我会使用 Google Gemini 模型（或 OpenAI GPT 模型，根据你的实际情况说明）。请为我写出一个完整的 Python 脚本，要求：
> 
> - 使用 `langextract` 库
> 
> - 输入文本指定为 `input_text` 变量（我将自行粘贴我的文本）
> 
> - 输出结果保存为一个 JSONL 文件，并生成一个交互式HTML高亮页面以供初步验证
> 
> - 请添加详细注释，指导我如何安装依赖和运行

**里程碑 2 的成功输出：** 一段可执行的 Python 脚本，你运行后得到一个 `extraction_results.jsonl` 和高亮 HTML。你会首次看到提取出的实体和关系，以及它们的原文位置。

**检查点：** 打开高亮页面，确认实体是否被正确标出。如果漏提或错提，回到里程碑 1 调整示例，让智能体帮你微调。

## 里程碑 3：将提取结果转化为图数据

**目标：** 将 LangExtract 的原始输出转换成 `nodes` 和 `links` 的 JSON，为 3D 可视化做准备。

**沟通指令：**

> 现在我有了 `extraction_results.jsonl`，里面每条记录包含 extraction_class、extraction_text、char_interval 和 attributes。
> 
> 我需要你帮我写一个 Python 脚本，将 JSONL 转换为图数据结构，要求如下：
> 
> - 每一个被提取的实体（如人物、动物等）都成为一个节点。节点 id 用实体文本本身（去重，若同名但语境不同可稍加区分）。
> 
> - 节点需携带属性：类别、原文位置区间、以及所有 attributes。
> 
> - 对于关系类提取（假设你的模板中定义了 relationship 或 causal_chain 类别），请从提取文本中解析出“因实体”和“果实体”。由于我们已要求关系格式为“(因) A —[关系]→ (果) B”，你可以用简单的字符串分割或正则提取 A 和 B。
> 
> - 如果实体节点还不存在，则自动创建。
> 
> - 输出一个 `graph_data.json`，结构为：  
>   `{ "nodes": [ { "id": "A", "group": "person", "char_interval": [0,12], ... }, ... ], "links": [ { "source": "A", "target": "B", "relation": "caused", "description": "放飞蝴蝶" }, ... ] }`
> 
> - 请确保属性中保留原始的 `char_interval`，方便后续点击回溯原文。
> 
> 请写出完整脚本，并告知我如何运行。

**里程碑 3 的成功输出：** 一个 `graph_data.json`，你的星球所需的所有“星体”和“引力线”已准备就绪。

## 里程碑 4：3D 星球可视化核心

**目标：** 获得一个可旋转、可点击展开的 3D 球体HTML页面。

**沟通指令：**

> 现在我有一个 `graph_data.json`，格式如前述。请为我创建一个独立的 HTML 文件 `cosmos.html`，使用 3d-force-graph 库实现以下功能：
> 
> 1. 所有节点初始分布在一个半径可调的球面上。
> 
> 2. 可拖拽旋转、缩放整个球体。
> 
> 3. 节点按 group 着色（请帮我设计一套配色方案，区分人物、动物、情感等类别）。
> 
> 4. 节点大小可反映连接数或重要性。
> 
> 5. **核心交互**：点击一个节点时，动态展开它的直接邻居（1-hop）。如果邻居节点尚未在图上显示，则即刻添加到球面上，并建立连线。再点击已展开的节点，可以折叠或继续展开它的下一层邻居，以此实现“无限链接”。
> 
> 6. 鼠标悬停节点时显示标签和详细信息。
> 
> 7. 最重要的是：点击节点或连线时，**必须展示原始文本的证据**。即利用节点携带的 `char_interval`，设计一个弹出窗或侧边栏，显示原文片段并高亮对应文字。请用 JavaScript 实现此功能。
> 
> 请写出完整的 HTML 文件，包含所有 CSS 和 JS，可以直接在浏览器中打开。数据可以直接内嵌在脚本中（请将 graph_data.json 的内容赋值给一个变量）。

**额外加分指令：**

> 为了让星球更生动，请加入：
> 
> - 节点周围有微弱的光晕
> 
> - 连线带动态流动粒子，表示因果方向
> 
> - 背景为星空

**里程碑 4 的成功输出：** 一个让你惊叹的 `cosmos.html`，你可以在浏览器里旋转、点击，看到因果链无限延伸。

## 里程碑 5：装配、测试与美化

**目标：** 将所有步骤串联，微调视觉和交互，并实现从“星球节点”一键跳转回“原文高亮页面”的终极闭环。

**沟通指令：**

> 我们已经有了：
> 
> - `extraction_results.jsonl` 和对应的交互式高亮 HTML（由 LangExtract 生成）
> 
> - `cosmos.html` 3D 星球
> 
> 请帮我完成最后的集成优化：
> 
> 1. 修改 `cosmos.html`，在节点详情弹窗中，除了显示原文高亮片段，再添加一个“定位原文”按钮，点击后在新窗口打开高亮 HTML 并自动滚动到对应位置（可利用 URL 锚点）。
> 
> 2. 审核整个流程，编写一个自动化脚本 `run_all.sh`（或 `run_all.bat`），依次执行提取→转换→可视化，让后续更新文本后一键重生星球。
> 
> 3. 对我的 `graph_data.json` 进行质量检查，如果有孤立节点或重复，请自动清洗。
> 
> 4. 最后，请根据我的文本主题，设计一套更贴合的配色和星球风格（例如科技蓝、或复古羊皮纸色调）。

**里程碑 5 的成功输出：** 一个完整的、可交付的“文本数据星球”项目包。

## 与智能体沟通的黄金法则

1. **永远提供具体样例**：当你说“我想要无限展开”，请紧接着描述一个具体场景：“比如点击‘蝴蝶’，立刻出现‘扇动翅膀’、‘花粉传播’等节点，再点击‘扇动翅膀’，又出现‘起风’等。”

2. **要求可视化而非解释**：不要说“给我解释如何实现”，而要说“请直接写出可运行的代码”。

3. **分步交付**：如果它给的代码报错，将错误信息贴回去，它会修正。

4. **保护你的数据**：如果文本敏感，可用假名或虚构文本先跑通流程，后续替换真实文本。

5. **记录迭代**：每次成功的里程碑，把智能体给你的代码和配置保存好，这样随时可以复活星球。
